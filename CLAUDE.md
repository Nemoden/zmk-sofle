# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

ZMK firmware configuration for two split keyboards:
- **Eyelash Sofle** вҖ” nRF52840, Nice View displays, encoder, RGB, joystick
- **TOTEM** вҖ” Seeeduino XIAO BLE, 38-key column-staggered choc

Both share the same logical 38-key layout, behaviors, and macros. The Sofle has extra hardware keys (number row, encoder, joystick) that are mapped to `&none`.

## Build

Firmware builds via GitHub Actions вҖ” push to trigger. No local build needed.

- **CI workflow**: `.github/workflows/build.yml` delegates to `zmkfirmware/zmk` reusable workflow
- **Targets**: defined in `build.yaml` вҖ” Sofle left/right, TOTEM left/right, Studio variant
- **Keymap SVGs**: auto-generated by `.github/workflows/draw.yml` on config changes, committed to `keymap-drawer/`

## Architecture

```
config/
  eyelash_sofle.keymap  вҖ” Sofle keymap (64 positions, 38 used)
  totem.keymap           вҖ” TOTEM keymap (38 positions)
  behaviours.dtsi        вҖ” shared: hold-tap, mod-morph, tap-dance, sticky keys
  macros.dtsi            вҖ” shared: tmux, chrome, brackets, operators
  eyelash_sofle_keypos.h вҖ” Sofle position defines (KEYS_L, KEYS_R, THUMBS)
  totem_keypos.h         вҖ” TOTEM position defines (same macro names, different values)
  *.conf                 вҖ” board-specific Kconfig (RGB, encoder, sleep, BT)
  west.yml               вҖ” west manifest (ZMK, eyelash_sofle board module, mario animation)

boards/
  arm/eyelash_sofle/     вҖ” custom board definition (nRF52840, matrix, displays, encoder)
  shields/totem/         вҖ” TOTEM shield (copied from GEIGEIGEIST/zmk-config-totem)

zephyr/module.yml        вҖ” declares board_root for ZMK to find boards and shields
```

## Key Design Decisions

- **Shared behaviors/macros**: both keymaps `#include` the same `behaviours.dtsi` and `macros.dtsi`. Key position macros (`KEYS_L`, `KEYS_R`, `THUMBS`) are defined per-board in separate headers but with identical names, so `hml`/`hmr` homerow mods resolve correctly for each board.
- **Conditional tri-layer**: holding layer 1 + layer 2 keys activates layer 3 (SYSTEM/BT).
- **Layer-tap on RH0**: `&lt 2 BACKSPACE` on all layers вҖ” tap=backspace, hold=layer 2.
- **Mod-morphs**: `morph_bslh`/`morph_fslh` suppress shift to always produce `\`/`/`. `morph_pgup_b`/`morph_pgdn_f` produce Alt/Ctrl+b/f when modifiers held.
- **No encoder/RGB/joystick on TOTEM**: `totem.conf` strips those configs.

## Layers

| Layer | Name | Purpose |
|-------|------|---------|
| 0 | LAYER0 | QWERTY base |
| 1 | CHORD-NAV | Sticky mods, arrows, tmux/chrome nav, PG_UP/DN |
| 2 | SYN-NUM | Symbols (left), numpad (right), brackets with auto-close |
| 3 | SYSTEM | Bluetooth profiles (tri-layer: 1+2) |

## Flashing

- **Sofle**: flash left half only (wired split, right gets firmware via connection)
- **TOTEM**: flash left half only (wireless split, right half auto-pairs and forwards keypresses; keymap runs on left/central side)
- **XIAO BLE bootloader**: double-tap reset, `cp -X firmware.uf2 /Volumes/XIAO-SENSE/`. macOS will report error -36 вҖ” ignore it, firmware flashes successfully (device disconnects mid-copy)

## ZMK Version

Both `west.yml` and `.github/workflows/build.yml` must pin to the same ZMK version (currently `v0.3.0`). Using `@main` will break builds due to Zephyr 4.1 board naming changes.

## When Editing Keymaps

- Binding count must match matrix: **64** for Sofle, **38** for TOTEM. Use the python check:
  ```python
  # Count bindings including _v_ and XXX macros
  re.findall(r'(&\w+|_v_|XXX)', layer_text)
  ```
- `_v_` = `&trans`, `XXX` = `&none`
- Changes to layers/behaviors should be applied to **both** keymaps to keep them in sync.
- `keymap_drawer.config.yaml` controls SVG visualization вҖ” update when adding new bindings/behaviors.
